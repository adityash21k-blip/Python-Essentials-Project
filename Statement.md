# Hostel Room Allocation System 

## Problem Statement
In many colleges and hostels, room allotment is still done manually using notebooks or spreadsheets. This creates challenges such as difficulty in tracking available rooms, human errors, duplicate entries, confusion during peak admission periods, and no proper record of the date of room allotment. Therefore, there is a need for a simple digital system that can store room and student details, allocate rooms automatically, and maintain accurate records.

## Scope of the Project
This project is designed to manage the basic hostel activities related to room allocation. The system allows the admin to add rooms, add students, view all records, allocate rooms based on gender and room capacity, deallocate rooms when needed, and clear all stored data. The project works through a command-line interface using Python and focuses only on the room allocation process. It does not include fees management, attendance monitoring, warden login, or online portal features.

## Target Users
- Hostel wardens
- Hostel management staff
- Administrative officers
- Student accommodation coordinators

## High-Level Features
- Add and store room details with capacity and gender.
- Add and store student information.
- View list of rooms with current occupancy.
- View list of students with allocation status.
- Automatic room allocation based on gender and vacancy.
- Store date of room allocation.
- Remove a student from a room.
- Clear all stored data on admin command.
